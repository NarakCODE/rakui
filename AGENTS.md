## Workflow

- Use plan mode for non-trivial tasks.
- Write the plan in `tasks/todo.md` before coding.
- Re-plan if needed.
- Verify before marking work complete.

## Task Tracking

- Keep `tasks/todo.md` updated.
- Mark progress as you work.
- Add short results per task.

## Lessons

- Log mistakes in `tasks/lessons.md`.
- Include both cause and prevention.

## Standards

- Keep solutions simple.
- Minimize changes.
- Fix root causes.

## Verification

- Run tests when relevant.
- Check behavior and edge cases.
