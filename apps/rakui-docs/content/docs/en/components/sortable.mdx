## Layout [#layout]

Import the parts you need and compose them together.

```tsx title="sortable-usage.tsx"
import {
  Sortable,
  SortableContent,
  SortableItem,
  SortableItemHandle,
  SortableOverlay,
} from "@/components/ui/sortable";

return (
  <Sortable value={items} onValueChange={setItems}>
    <SortableContent>
      {items.map((item) => (
        <SortableItem key={item.id} value={item.id}>
          <SortableItemHandle />
          {item.title}
        </SortableItem>
      ))}
    </SortableContent>

    <SortableOverlay />
  </Sortable>
);
```

## Usage [#usage]

### With Primitive Values [#with-primitive-values]

When the list contains primitive values, the items themselves are used as sortable identifiers, so `getItemValue` is optional.

<ComponentTabs name="sortable-primitive-values-demo" />

```tsx title="sortable-example.tsx"
const [items, setItems] = React.useState(["Inbox", "In Review", "Done"]);

<Sortable value={items} onValueChange={setItems}>
  <SortableContent className="flex flex-col gap-2">
    {items.map((item) => (
      <SortableItem
        key={item}
        value={item}
        asHandle
        className="rounded-lg border bg-card p-3 shadow-xs"
      >
        {item}
      </SortableItem>
    ))}
  </SortableContent>
</Sortable>
```

### With Object Arrays [#with-object-arrays]

When the list contains objects, pass `getItemValue` so the component can derive a stable sortable id.

<ComponentTabs name="sortable-demo" />

```tsx title="sortable-example.tsx"
const [tasks, setTasks] = React.useState([
  { id: "write-copy", title: "Write landing page copy" },
  { id: "review-pr", title: "Review sortable pull request" },
  { id: "ship-demo", title: "Ship docs demo" },
]);

<Sortable
  value={tasks}
  onValueChange={setTasks}
  getItemValue={(item) => item.id}
>
  <SortableContent className="flex flex-col gap-2">
    {tasks.map((task) => (
      <SortableItem
        key={task.id}
        value={task.id}
        className="rounded-lg border bg-card"
      >
        <div className="flex items-center gap-3 p-3">
          <SortableItemHandle className="rounded-md border p-2">
            <GripVertical className="size-4" />
            <span className="sr-only">Reorder {task.title}</span>
          </SortableItemHandle>
          <span>{task.title}</span>
        </div>
      </SortableItem>
    ))}
  </SortableContent>
</Sortable>
```

### Horizontal Lists [#horizontal-lists]

Set `orientation="horizontal"` to restrict movement to the x-axis and switch the default sorting strategy to the horizontal list preset.

<ComponentTabs name="sortable-horizontal-demo" />

```tsx title="sortable-snippet.tsx"
<Sortable
  value={items}
  onValueChange={setItems}
  orientation="horizontal"
>
  <SortableContent className="flex gap-3">
    {items.map((item) => (
      <SortableItem
        key={item}
        value={item}
        asHandle
        className="min-w-40 rounded-lg border bg-card p-4 text-center shadow-xs"
      >
        {item}
      </SortableItem>
    ))}
  </SortableContent>
</Sortable>
```

### Custom Overlay [#custom-overlay]

Use `SortableOverlay` when the dragged preview should differ from the source element.

<ComponentTabs name="sortable-demo" />

```tsx title="sortable-snippet.tsx"
<SortableOverlay>
  {({ value }) => {
    const activeTask = tasks.find((task) => task.id === value);

    if (!activeTask) return null;

    return (
      <div className="w-80 rounded-xl border bg-card p-3 shadow-lg">
        {activeTask.title}
      </div>
    );
  }}
</SortableOverlay>
```

## Accessibility [#accessibility]

Sortable includes default screen-reader announcements and keyboard instructions for pick up, move, drop, and cancel interactions. For the clearest accessible UX, prefer a real `SortableItemHandle` button with an `sr-only` label when only part of the item should start dragging.
