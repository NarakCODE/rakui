## Layout

Import the parts you need and compose them inside the `SpeedDial` root.

```tsx title="speed-dial-demo.tsx"
import {
  SpeedDial,
  SpeedDialTrigger,
  SpeedDialContent,
  SpeedDialItem,
  SpeedDialLabel,
  SpeedDialAction,
} from "@/components/ui/speed-dial";

export function SpeedDialDemo() {
  return (
    <SpeedDial>
      <SpeedDialTrigger>
        {/* Trigger icon */}
      </SpeedDialTrigger>
      <SpeedDialContent>
        <SpeedDialItem>
          <SpeedDialLabel>Label</SpeedDialLabel>
          <SpeedDialAction>
            {/* Action icon */}
          </SpeedDialAction>
        </SpeedDialItem>
      </SpeedDialContent>
    </SpeedDial>
  );
}
```

### Fixed Positioning

To pin the speed dial to a corner of the viewport, apply `fixed` positioning classes to the **`SpeedDial` root**, not the trigger.

<ComponentTabs name="speed-dial-fixed-demo" />

```tsx title="speed-dial-snippet.tsx"
<SpeedDial className="fixed right-4 bottom-4">
  {/* trigger and content */}
</SpeedDial>
```

<Callout>
  **Important:** Always apply fixed or absolute positioning to the `SpeedDial` root container, not to `SpeedDialTrigger`. The trigger is positioned relative to the root, and the content is positioned relative to the trigger.
</Callout>

## Features

- **Composable architecture** — Build the speed dial from individual, composable parts.
- **Click or hover activation** — Trigger the menu with a click or by hovering over the button.
- **Four expansion directions** — Expand toward `top`, `right`, `bottom`, or `left`.
- **Staggered animations** — Action items animate in and out with calculated delays based on their order.
- **Keyboard navigation** — Full keyboard support including `Escape` to close and `Tab` to exit.
- **Outside click detection** — Automatically closes when clicking outside, with touch support.
- **Controlled or uncontrolled** — Manage state internally or externally.
- **Accessible** — Uses `role="menu"`, `aria-expanded`, `aria-controls`, and `aria-labelledby` for screen reader support.
- **Slot support** — All parts support `asChild` for custom element rendering.

## Usage Notes

- **Fixed positioning** — Apply `fixed` or `absolute` positioning to the `SpeedDial` root, not the `SpeedDialTrigger`. The trigger is positioned relative to the root, and the content is positioned relative to the trigger.
- **State management** — The component uses an internal store pattern (`useSyncExternalStore`) for state management. All sub-components access the same store through context.
- **Hover timers** — In hover mode, a close timer (`DEFAULT_HOVER_CLOSE_DELAY = 100ms`) and an open delay (controlled by `delay` prop) prevent accidental open/close cycles.
- **Animation delays** — Each `SpeedDialItem` receives a calculated animation delay based on its position in the children array. Items animate in sequentially during open, and in reverse order during close.
- **Unmount timing** — When closed, the content remains in the DOM long enough for all animations to complete before unmounting (unless `forceMount` is set).
- **Interact outside** — The `onInteractOutside` callback receives a custom event with the original `PointerEvent`. Call `event.preventDefault()` to prevent the speed dial from closing.
- **Node registration** — Action elements register themselves in a `Map` via refs, enabling keyboard navigation and outside click detection to work correctly.

## Accessibility

- `SpeedDialTrigger` renders with `role="button"`, `aria-expanded`, `aria-controls`, and `aria-haspopup="menu"`.
- `SpeedDialContent` renders with `role="menu"` and `aria-orientation`.
- `SpeedDialAction` renders with `role="menuitem"` and `aria-labelledby` pointing to its associated label.
- Pressing `Escape` closes the menu and returns focus to the trigger.
- Pressing `Tab` at the first or last focusable element closes the menu.
- The component respects `prefers-reduced-motion` by using CSS transitions that can be overridden.
